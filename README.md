## Avance
Las ramas en Git permiten trabajar en diferentes cambios del proyecto sin afectar el código principal. Cada rama funciona como una copia independiente donde se pueden desarrollar nuevas funciones, corregir errores o realizar pruebas.


En nuestro proyecto utilizaremos una rama principal llamada main, donde estará la versión estable del sistema.

Cada integrante creará ramas secundarias para trabajar sus tareas de manera independiente. Por ejemplo:

feature-login
feature-vehiculos
feature-pagos
fix-errores

Cuando una funcionalidad esté terminada y probada, los cambios se unirán a la rama main mediante un merge.

De esta manera podremos trabajar en equipo de forma más organizada, segura y eficiente.
